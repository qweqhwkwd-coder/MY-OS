## Провайдеры (бесплатные лимиты)

| Провайдер | Модель | Лимит | Для чего |
|---|---|---|---|
| Google Gemini | 2.5 Flash | 1500 req/day | Основной AI (текст, vision, анализ) |
| Groq | Llama 70B | 1000 req/day, 30 RPM | Быстрый парсер голоса (~315 tok/s) |
| Anthropic Claude | Haiku/Sonnet | платно | Опционально, лучшее качество |

Распределение: голос→интент и классификация → Groq; фото еды, анализ питания, AI-тренер, недельный отчёт → Gemini.

## Сценарии (целевые)

1. **Фото этикетки/блюда** → Gemini Vision → КБЖУ + комментарий нутрициолога → сохранить в лог.
2. **Голос → действие** → faster-whisper транскрибирует → Groq определяет интент (модуль/действие/данные) → распределяет по модулям. Перед сохранением бот показывает «Я правильно понял?».
3. **Голосовое планирование дня/недели** → разбор на тренировки/встречи/задачи/дневник.
4. **AI-анализ питания** в контексте дня+тренировки+нормы.
5. **Недельный отчёт** (авто вс 20:00) — сбор данных за 7 дней, генерация отчёта.

## Structured Outputs

Оба провайдера поддерживают гарантированный JSON. Использовать `response_format={"type":"json_object"}` + Pydantic-модель для парсинга интента. Без этого — баги парсинга.

## Узлы, решить ПЕРЕД внедрением (из ROADMAP §5)

1. **faster-whisper и 512MB RAM.** Модель `medium` не влезает в free Render. Варианты: мелкая модель, готовый Whisper API, или Render Starter $7. Модель вшивать в Docker-образ.
2. **Лимит Supabase Storage ~1GB.** Нужна политика автоудаления старых медиа (фото еды/голос).
3. На MVP КБЖУ считается **формулой**, без AI — AI только когда дойдём.

## Когда подключать Claude

Если качества Gemini/Groq не хватит для конкретной задачи (AI-тренер, сложный анализ) — подключить Claude точечно для этой задачи. Архитектура позволяет менять провайдера на задачу независимо.
