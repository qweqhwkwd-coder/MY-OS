# MY-OS — Технический отчёт для передачи (handoff)

> Дата среза: после полного аудита кода/доков (см. историю коммитов).
> Bot-first MVP, который успел вырасти: Telegram-бот + FastAPI-бэкенд +
> Supabase (PostgreSQL) + **Mini App (React, уже существует и развивается)**,
> задеплоено на Render (free tier) + GitHub Pages.

---

## 2. Стек и зависимости

**Backend:** Python 3.12, FastAPI + aiogram 3 + pydantic-settings + supabase-py
(синхронный клиент, вызывается внутри async-обработчиков — см. «Известные проблемы»).

**Mini App (`miniapp/`):** React 19 + Vite + Tailwind v4. Целевой стек из продукта
(TanStack Query, Zustand, shadcn/ui, Recharts) — НЕ внедрён, сейчас обычный
`useState`/`useEffect`. Деплой статикой на GitHub Pages, открывается кнопкой
из бота (`WebAppInfo`).

**AI:** НЕ подключён.

**Хостинг:** backend — Render, Docker Web Service, free tier; Mini App — GitHub Pages.

## 3. Состояние по модулям

Подробная таблица — `docs/claude/current-state.md` (она же — источник правды).
Кратко: из 17 модулей продукта **12 реализованы хотя бы в базовом виде**
через бота (задачи, ритуалы+вода, питание, дневник, цели, идеи, встречи, сон,
тренировки, финансы, замеры, inbox, колесо баланса, дайджест), Mini App
покрывает 5 экранов (Today/Water/Rituals/Tasks/Food) + общий профиль с
рангами/HP/стриком. RPG-система: 8 статов, level, ранги (10 тиров), HP
(`calculate_hp`), стрик (`get_streak`) — всё это уже в коде, не только в плане.

Не реализовано: AI-слой, уведомления/планировщик, полный 5-шаговый онбординг,
большая часть «Настроек», мультипользователь (initData→JWT мост), физ-профиль/КБЖУ.

**⚠️ Язык интерфейса фактически украинский**, не русский, как было задокументировано
ранее (и в боте, и в Mini App). Сверить с автором, какой язык целевой.

## 4. Переменные окружения (только названия)

| Переменная | Обязательна | Назначение |
|---|---|---|
| `BOT_TOKEN` | да | токен бота от @BotFather |
| `SUPABASE_URL` | да | URL проекта Supabase |
| `SUPABASE_SERVICE_ROLE_KEY` | да | секретный ключ, обходит RLS — бэкенд работает под ним |
| `SUPABASE_ANON_KEY` | нет (default "") | публичный ключ, пока НЕ используется в коде |
| `WEBHOOK_BASE_URL` | нет (default "") | публичный адрес сервиса на Render; если пусто — вебхук на старте НЕ ставится |
| `WEBHOOK_SECRET` | нет (default "") | секрет вебхука, отдельный от токена; если пусто — генерируется случайно на каждом старте |
| `PORT` | нет (default 8000) | порт; на Render подставляется автоматически |

Значения нигде в репозитории не хранятся; `.env` в `.gitignore`.

## 5. Запуск

**Продакшен backend (Render, через Docker):** без изменений — см. `docs/claude/architecture.md`.

**Mini App:** собирается и деплоится на GitHub Pages отдельно от backend
(`miniapp/` — самостоятельный Vite-проект, не часть Docker-образа backend).

**Миграции БД (вручную):** Supabase → SQL Editor → выполнить файлы из
`db/migrations/` по порядку, КАЖДЫЙ отдельным запуском (включая `014` —
без неё `add_xp`/`get_user_stats` упадут на вызове RPC, которой нет в БД).

**Локальный запуск backend:** см. `docs/claude/architecture.md` — без изменений
(только вебхук, нужен туннель для локальной разработки бота).

## 6. Известные проблемы / TODO (актуально на момент среза)

1. **Авторизация только под одного пользователя.** `service_role` обходит RLS; моста initData→JWT нет.
2. **Синхронный `supabase-py` в async-обработчиках.** Блокирует event loop. Для одного юзера ок.
3. **Нет отмены воды.** `add_water` только прибавляет.
4. **Нет планировщика / уведомлений / дайджеста-по-расписанию.** Дайджест есть только по запросу (`/digest`).
5. **Холодный старт Render free** (~30–60с после простоя).
6. **Нет автотестов.** Только `python -m py_compile`.
7. **Нет таблицы `stats_history`** → нет графиков динамики статов в Mini App.
8. **Миграции применяются вручную**, строго по одной; `002_add_stats.sql` потерян на диске (см. §1).
9. **Расхождение языка интерфейса** — украинский по факту, документация раньше утверждала русский.
10. **Полный 5-шаговый онбординг и физ-профиль/КБЖУ** не реализованы — только лёгкий `WelcomeScreen`.
11. **AI-слой** не подключён (Фаза 4 по `roadmap.md`).
12. **Mini App покрывает не все модули** — только 5 экранов ядра, остальные 7+ реализованных модулей доступны только через бота.

Пофикшено с прошлой версии этого отчёта (раньше значилось как открытый баг):
- ~~Фарм XP на ритуалах~~ — `toggle_ritual` больше не удаляет лог при снятии отметки.
- ~~Секрет вебхука выводится из токена~~ — теперь отдельная переменная, не производная.
